📖 简介

阅读T 是基于 Legado 的增强版分支，由明月照大江维护。

在保持原版核心阅读能力的前提下，重点补齐了以下方向：

  • ✨ 定时任务与自动化脚本
  • 🛠️ 功能强大的代码编辑器
  • 🚀 JS 运行环境与规则兼容性
  • 📱 其他阅读细节体验优化

🎯 核心能力

1️⃣ 定时任务自动化

功能强大的定时任务执行

  • ⏰ 支持 Cron 表达式调度脚本
  • 📋 支持任务日志查看（最近一次完整执行日志）
  • 🔗 支持一键导入：/auto?src= 与 /autoTask?src=
  • 📚 内置多书源自动检测更新，实时推送通知

2️⃣ 规则编辑器增强

专业级代码编辑体验

  • 🎨 WebView 代码编辑器，支持自动补全、语法校验、代码折叠、搜索替换、格式化等
  • 🔄 支持 CURL 转 AnalyzeURL
  • ⚡ 大文本场景下优化性能与稳定性
  • 🎯 语法错误实时检查，提升JS编写效率

3️⃣ JS 运行环境

更强大的脚本执行能力

  • 🔧 已切换 Rhino 1.8.1 到 htmlunit-core-js（4.21 分支）
  • 🔐 为规则运行环境启用 CryptoJS
  • 📤 java.get/post/head 增加 JSON 字符串请求头重载
  • 🌐 提升规则兼容性，支持更多复杂场景

4️⃣ 其他增强

细节体验持续优化

  • 🖼️ 图片点击事件支持
  • 📝 段评能力增强
  • 🌐 代理支持socks5服务器认证
  • 🔗 URL参数新增字段timeout、followRedirects、resolveIp
  • 🐛 Android 16 兼容性修复

❓ 常见问题

Q1：阅读T和原版阅读有什么区别？

A1：阅读T是基于 Legado 的增强分支，核心功能保持一致，主要增加了以下特性：

  • 定时任务自动化能力
  • 功能更强大的规则编辑器
  • 更好的 JS 运行环境兼容性
  • 各项细节体验优化

Q2：能否兼容原版阅读的数据？

A2：可以兼容。阅读T使用新的包名和签名，不会覆盖原版数据。首次使用建议先备份原版数据，再导入到阅读T中。

Q3：如何反馈问题或提出建议？

A3：欢迎通过以下方式反馈：

  • GitHub Issues：skybbk1001/legado/issues
  • 在本文下方留言

Q4：定时任务支持哪些功能？

A4：定时任务支持：

  • Cron 表达式调度
  • 书籍详情页一键添加自动检测
  • 自定义脚本执行
  • 任务日志查看
  • 一键导入分享
